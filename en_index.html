What is EasyVK?

EasyVK is a module for node.js, written in JavaScript, which is able to easily handle VKontakte API without registration.
The author of this module is javascript developer Kirill Novak (very modestly).

With EasyVK you can create applications on node.js any of type. Started from the simplest: searching likes in group, and ending more hard, such as creating bots or just news targeting.
EasyVK is written in order for the VK API to have support on the node.js, absolutely all EasyVK components are free and available with open source code.

At the moment EasyVK supports the main platforms VKontakte API

  • Callback API - group management, creating bots for the group
  • LongPoll - creating bots, listening to user or group account events
  • Streaming API - Targeting, collecting information about news, comments and other by certain rules
  • Methods - Standard methods for managing an account / group, and not only
  • Files - uploading files to the server VK, for example - cover for a group

What is necessary

In order to use EasyVK, make sure that you have installed node.js >= 8.* version

You must install the easyvk module
To do this, you must install the npm package manager and enter the command
npm install easyvk

After this you must create file index.js in the current directory.
In it, you need to put the this code 

const easyVK = require("easyvk");

All the code in this documentation was tested on version 0.4 You can be sure that for this version everything works stably. During the tests, many bugs and bugs were fixed, thanks to which I am now sure that this is the most stable release.

Let's start #backtotop

Before you start using EasyVK, you must understand how does he works.
EasyVK is primarily an intermediary between the VK server and your ideas. Since EasyVK works with VKontakte methods, we follow their requirements.
That's why before you start working with EasyVK, you must first log in and create an easyVK session.

In EasyVK there are several ways to authorize

  • 1. Authentication with user's access_token
  • 2. Authentication with group's access_token
  • 3. Authentication with user's username and password

What you need is up to you.
If you need to work with a group, then add group's access_token, but if you work with a personal account or user account - you have the right to choose how to authorize it. If you only need for personal use and you do not want to bother getting a token, select 3-d way.

So, here is the code that is needed to authorize with access_token 

              
errHandler = (err) => {console.log(err);}

easyVK({
    access_token: "your_token"
}).then((vk) => {
    console.log(vk.session.group_id) // For groups
    console.log(vk.session.user_id) // For user
}, errHandler);

And for user account we select this way: 


easyVK({
    username: "your_email@mail.ru_or_number_phone",
    password: "your_password"
}).then((vk) => {}, errHandler);

You might want to know what parameters can be used in the initialization object.
It's very simple, take a look at the this table.

Parameters for initialization object
Parameter Description Default
api_v VKontakte API version, which will be used in all queries. 5.73
access_token Access token to authorize and send API requests. Required parameter if session file is empty or no other data is entered for authorization. Absent
username User name. Need when authorizing a user. A mandatory parameter if the session file is empty or no other data is entered for authorization. Absent
password The user's password. Required only if the parameter username is not empty. Absent
reauth In case you need to overwrite the session file with new parameters or you do not want to be authorized for the saved data, you need to insert this parameter. false
save_session If there is a need not to save the session file, then you can set this parameter as false true
clean_session If you need to clear the session file after authorization, you can set this parameter as true false
session_file The path to the session file where your session will be saved. .vksession
code In the event that your account uses two-factor authentication, you must send the code from the SMS or code from the application that you use to generate this code. Absent
captcha_key If a CAPTCHA error occurs, then you are suspected of being a DDOS attack. Enter the CAPTCHA string in this parameter, which you will find in the error, if you print it correctly. ""
captcha_sid In this parameter you need to enter the captcha sID, which you get from an error. ""
This is the final list of authorization parameters. It is unlikely that it will be replenished, but anything can happen.

So, after you're logged in, a session file will be created that will contain a group's / user's session.
n this session some parameters are recorded: group_id, access_token, user_id, first_name, last_name, group_name, group_screen (litteral group's id).
In subsequent authorization attempts, all data will be taken from this file, even if they differ from those that you entered. If you do not want this, set the reauth as true

Methods#backtotop

There are several methods in the EasyVK object that you can use to work with the VKontakte API. The documentation on them can be read below.


vk.call(...)

This method refers to standard methods VKontakte API.
You can monitor the request by sending certain parameters to the server, and then receive a response. In case of an error, you can catch it and display it in the console without stopping the program.

Accessing this method from the current object ensures that the request goes with access_token from the session. In the case, if you need to make a request to a method that does not require authorization, you can use static methods, which will be discussed below. But this is optional.



Characteristic Value Description
@returns Promise Standard Promise object
@param methodName {String} Method name
@param data {Object} An object with parameters for the query, for example:
{text: "Hello!"}
@param methodType {String} One way to send a request:
["post", "get", "delete", "put"]

After a cal, in the resolve function will send the complete VK response is sent as an object.


Usage example

I'll give you a very simple example - sending a message. I ask you to note that I'm not referring to the easyVK function, but to the vk object that I already received after authorization. 



vk.call("messages.send", {
    user_id: "356607530",
    message: "Hello, Kirill! I am using your module EasyVK!",
    attachment: ["photo100172_166443618", "video85635407_165186811_69dff3de4372ae9b6e"]
}).then(vkr => {console.log(vkr)}, errHandler);

Or, for example - sending too much of a post. With this restriction, there are often many programmers, which is why I added support POST queries. 



vk.call("wall.post", {
    message: `very large text..... more >> `
}, "post").catch(errHandler);

For upload files using it, i recommend using the existing uploader, and abandoning the vk.call() method.


vk.saveSession(...)

This method saves session file to the specified path in the file_session parameter.

Conservation occurs synchronously.

The method does not accept anything and nothing returns.

Usage example


vk.session.foo = "bar";
vk.saveSession();

After the next run, a variable vk.session.foo will be available to you.So you can cache any data for yourself. You will receive this variable only if you do not ask to be reauthorized with new data.

Static methods#backtotop

Static methods - are methods that do not require object initialization.
In our case, these methods do not require authorization. In most cases, static methods in EasyVK are just additional functions that were applied during the development of EasyVK. These functions are available to each utility, which is why I use them.


Static methods can be used for the convenience of development, or, for example, to access VKontakte API methods that do not require authorization (the example will be below).
Well, now we will just analyze each of them in detail.


The standard way to use static methods 


easyVK.static.someMethodName();

static.call(...)

This method refers to the VKontakte API method without requiring authorization. Such a need arises often when the request does not require any access_token.
EasyVK does not require you to know exactly whether authorization is required for a method or not. But also EasyVK does not know if authorization is needed, or not.
In the event that you are accessing this method with the intention of executing a request that requires authorization, you will receive an error not from EasyVK, but from the VKontakte server.
Thanks to this, the library can live for a long time, and it will be necessary to support it only if the API architecture changed or, for example, a new platform is added.

So, a little description.

Characteristic Value Description
@returns Promise Standard object Promise
@param methodName {String} Method name
@param data {Object} An object with parameters for the query, for example:
{text: "Hello!"}
@param methodType {String} One way to send a request:
["post", "get", "delete", "put"]

Usage example


easyVK.static.call("users.get", {
    user_ids: ["1", "156"]
}).then((vkr) => {
    console.log(vkr.response[0].id); // 1
}, (e) => {console.log(e);});

static.urlencode(...)

This method serializes the object into a string as parameters in the URL for the query.
Everything is very simple.

Characteristic Value Description
@returns String Returns an object serialized to a string. Example: "foo=bar&foo2=barNew"
@param object {Object} The object to be serialized, maximum is 1 level.

Usage example


const request = require("request");
const easyVK = require("easyvk");

let objGetData = {
    message: "Hello",
    user_id: "356607530",
    access_token: "your_token",
    v: "5.73"
}

let url = "https://api.vk.com/method/messages.send?";

url += easyVK.static.urlencode(objGetData);

request.get(url, (err, res) => {
   if (err) throw err;
   console.log(res.body); 
});

static.checkJSONErrors(...)

This function checks the VKontakte server's response to errors.
That is, for example - the server did not respond with a JSON response will be issued unexpected value {val} in JSON object at {pos} position an error. This can happen at any time.
Also this function checks your error request, and if the server respond with JSON that an error has occurred, this function will return false. If the query succeeds, the object (already parsed with JSON) will return the VKontakte's response. The example will be shown more clearly.

Characteristic Value Description
@returns {Object} or {Boolean} The server JSON response object, or, if an error occurred, the false value.
@param vkr {String} The string with the reply of the server VKontakte VKResponse
@param reject {Function} Reject function that will be called if an error occurs. This function receives the error description as the first parameter.

Usage example

In this example, a request is made to the VK server, but because of the slash after the method name, the server will respond to us with an HTML document in which it will be written 403 Forbidden.
There is no JSON, so there will be an error. 


const easyVK = require("easyvk");
const request = require("request");

let url = "http://api.vk.com/method/users.get/?user_ids=1&v=5.73";
request.get(url, (err, res) => {
    if (err) throw err;
    
    let json = easyVK.static.checkJSONErrors(res.body, (err) => {
        console.log(err);
    });

    if (json) {
        console.log(json.response[0].id); // not output, error will occur
    } else {
        console.log("invalid json", res.body);
    }
});

Use this method only when you are certain that the server should return JSON. In other cases, use your own solutions.

static.checkErrors(...)

This function will check for errors in your request. When the VKontakte server answers us with a valid JSON, which says that there is an error in the request, or, for example, the user is locked, and you are trying to get information about it, this function will return this error. It is configured for all kinds of query errors, including, and for CAPTCHA error: when the request requires a confirmation of the captcha.

Characteristic Value Description
@returns {String} Description of the query error, for example: you need an authorization confirmation code for two-factor verification.
@param vkr {Object} VKontakte server response object, already parsed JSON

Usage example


const easyVK = require("easyvk");
const request = require("request");

let url = "http://api.vk.com/method/users.get?user_ids=-1&v=5.73";
request.get(url, (err, res) => {
    if (err) throw err;

    if (!res.body) res.body = "{}";

    // Here it is more accurate. 
    // We make a request such that we are sure of it
    let vkr = JSON.parse(res.body); 
    
    let vkErr = easyVK.static.checkErrors(vkr);
    
    if (vkErr) {
        throw new Error(vkErr); // Will output Invalid user id error
    } else {
        console.log(vkr.response[0].id);
    }
});

Streaming API #backtotop

Streaming API — It is a tool for obtaining public data from VKontakte by the given keywords. (Read russian version)

That is, in simple terms: you can, by certain rules, search for information in the VK in real time (well, almost).
When someone puts out a post with a cat word, your server can receive this post thanks to the Streaming API immediately after the post is published. So it goes.
It works same way for a comments.
Information comes to you not from a certain group, but from the general mass of VKontakte's events. More information about the Streaming API itself you can read here (for all), and here for those who want to get acquainted with the technical side of the issue.

EasyVK supports Streaming API as a separate utility. You can use it both after authorization, and without. Below is an instruction for using the Streaming API methods and an example of using.

streamingAPI.connect(...)

In order to create a stream, you need to connect to the VK server by WSS:// protocol, you do not do this, you just need to enter the connection parameters, namely, the parameters of your application. After a successful connection, you will get the first parameter stream object: now you have access to the settings of your stream, you can add rules, delete; listen to events and so on. Let us examine more in detail below

In order to create your application, go to this link and select the Standalone type.

Characteristic Value Description
@returns {Promise} Standard object {Promise}, in the resolve function sends stream object
@param application {Object} Application settings, what parameters to enter in it - see below

In the function, you must pass an application object with settings of your application.

Application settings. Options
Option Description
clientSecret The string of the key of your application is taken from the application settings.
clientId The ID of your application.

Usage example


vk.streamingAPI.connect({
    clientSecret: "your_application_secret_key",
    clientId: "id_of_your_application"
}).then((stream) => {
    
}).catch(errHandler);

That's all. Now you have created your stream. Next, you will configure the rules and listen for events.

An example for those who do not need authorization (in fact, it is not needed). 


easyVK.streamingAPI.connect(/*...*/) 
// Without authorization. Works the same way, but with one exception - 
// for this method the debugger will not work, it will be fixed in 0.5 version
// usually a debugger is not needed here

stream.addRule(...)

This method adds a rule to the general rule list.

Characteristic Value Decription

@param tag {String} Tag rule's, key rule's
@param value {String} The rule itself.

Usage example


vk.streamingAPI.connect({/*application params*/}).then(stream => {
    stream.addRule("cats", "cat -dog").then((vkr) => {
        console.log(vkr);
    }, errHandler);

stream.deleteRule(...)

This method removes the rule from the general rule list.

Characteristic Value Description
@returns {Promise} Standard object Promise
@param tag {String} Rule tag, rule key.

Usage example


vk.streamingAPI.connect({/*application*/}).then(stream => {
    stream.addRule("cats", "cat -dog").then((vkr) => {
        stream.deleteRule("cats").then((vkr_deleted) => {
            console.log(vkr_deleted);
        }, errHandler);
    }, errHandler);
}).catch(errHandler);

stream.initRules(...)

Instead of using methods like stream.deleteRule() and stream.addRule(), i recommend using only this method.
This method is something like a manager of your rules. You write all the rules in one object and every time you change something in it, all the changes apply to the stream (only after the restart).
I.e: If you remove the rules from the object, they will be removed from the stream. If you change the rule, it will change. At the end you will get the change log. It will take out all the rules that you changed / deleted / added. Convenient thing, however.

Characteristic Value Description
@returns {Promise} Standard object Promise, in a resolve function sends change log, which came into force.
@param rules {Object} An object with rules where the pair key => value is a pair of tag => rule.

Usage example


vk.streamingAPI.connect({/*application*/}).then(stream => {
    stream.initRules({
        "cats": "cat",
        "dogs": "dog puppy"
    }).then((log) => {
        console.log(log);
    }).catch(errHandler);
}).catch(errHandler);

stream.deleteAllRules(...)

This method removes all rules from the general list of stream rules

Characteristic Value Description

Events

After initializing the rules and installing them, you can receive data from the stream. Posts are received on the stream, comments and shares. You can listen to all these events and create some statistics ... and in general, it's not my business. In general, you have the ability to receive data from a stream.

Since the stream object extends from EventEmitter, then it includes .on() and .emit() methods. These methods serve only to listen to the event and create it. And so, what are the events? At the moment these are all the events from the table below, take a look.

Event Event description
post Occurs when a user or group has posted a record on their wall.
comment Occurs when a user or group comments on someone's record (as I understand, only a record, but it's not accurate).
share Occurs when a user or group has share to the wall with a record.
failure Occurs when a critical error occurs on the server or when the connection is disconnected / closed.
error Occurs when there is some error during connection or during the transfer of information.

Usage example


vk.streamingAPI.connect({/*application*/}).then(stream => {
    stream.initRules({
        "cats": "cat"
    }).then(log => {
        
        stream.on("post", (event) => { // Listening to all post events
            console.log(event.event_url);
        });

    }).catch(errHandler);
}).catch(errHandler);

User LongPoll API #backtotop

VKontakte provides an opportunity to listen to events through long polls (requests) - LongPoll, we will not blame them, I think they know what they are doing. EasyVK supports this platform just like Streaming API, as a separate utility, tightly connected with the EasyVK object.
Documentation below.

longpoll.connect(...)

This function connects to the LongPoll (user) server, and then sends in a resolve function the LongPollConnection object.

Characteristic Value Description
@returns {Promise} Standard object Promise, in a resolve function sends the LongPollConnection object.
@param params {Object} LongPoll connection settings at different stages of the connecting. Usually this object is not needed, I analyze it in more detail in the longpoll.connect() (full) guide.

Usage example


vk.longpoll.connect().then((connection) => {
    console.log(connection);
}).catch(errHandler);

connection.addEventCodeListener(...)

Since in the User LongPoll API all events come in the form of an array, and the names of the events are essentially numbers, it did not make sense to add all the events from LongPoll, so I added only the basic events. And you can listen to everything else with the help of the add-on. Perhaps in the future, this method will change slightly and yet all events will be indicated, but so far. Dont be upset. Everything is fine.

Characteristic Value Description
@returns {Promise} Standard object Promise, resolve not being called (correct me if it's wrong).
@param code {Number} The event number, the list of all event codes, can be viewed here.
@param handler {Function} callback function that will be called when an event with a code occurs.

Usage example


connection.addEventCodeListener(3, (event) => { 
    // Resetting message flags, for example: readed
    console.log(event);
}).catch(errHandler);

connection.close(...)

This method breaks the connection to the LongPoll server and triggers an close event.

Characteristic Value Description
@returns {Promise} Standard object Promise, in a resolve function sends nothing.

Usage example


vk.longpoll.connect().then(connection => {
    connection.close(); // Close connection
}).catch(errHandler);

longpoll.debug(...)

This method allows you to add a debug function to which all VK server responses will be transmitted that relate only to connecting to LongPoll.

Characteristic Values Description
@returns LongPollConnection or false If the function is successfully added, it will return the LongPollConnection object, else will return false
@param debugg {Function} Debugger-function, all responses will be sent to it.

Usage example


vk.longpoll.connect().then(connection => {
    connection.debug((vkr) => {
        console.log(vkr); // not a JSON, regular response string
    }).on("message", (msg) => {
        console.log("MESSAGE", msg);
    });
}).catch(errHandler);

longpoll.connect(...) (full)

This is a more detailed documentation of the longpoll.connect() method.
As already mentioned, the method takes the params object. Now I will describe it.
In this object, the main two properties are objects: forGetLongPollServer, forLongPollServer

forGetLongPollServer - This is the object with the settings for the first request to the server - the method request messages.getLongPollServer. All the parameters that will be applied to this object in this object.

Usage example


vk.longpoll.connect({
    forGetLongPollServer: {
        lp_version: "2",
        need_pts: "1"
    }
}).then(connection => {
    console.log(connection.config.responseGetServer.response.pts);
}).catch(errHandler);

forLongPollServer - these are the parameters for requesting each Poll to the server. Here you can set an example, the wait parameter.
More details about these parameters can be read here

Usage example


vk.longpoll.connect({
    forLongPollServer: {
        wait: "1" // Дудосим крч :D
    }
}).then(connection => {
    console.log(connection.config);
}).catch(errHandler);

Events

Since LongPoll is primarily intended for listening to events, you can do this very easily in EasyVK. As well as Streaming API, the LongPollConnection object extends from the EventEmitter object. This way you can listen to server events. Look at the table of supported events.

Event code Event Event description
4 message Occurs when a user sends a message or a message is sent to him.
8 friendOnline Arises when a friend of the user is online.
9 friendOffline Occurs when a friend of the user is offline.
51 editChat Occurs when a user or someone changes the conversation settings.
61 typeInDialog Occurs when the user types a message in a dialog with user.
62 typeInChat Occurs when the user types a message in a chat (conversation).

Usage example


vk.longpoll.connect().then(connection => {
    connection.on("message", (msg) => {
        console.log(msg);
    });
}).catch(errHandler);

The rest of the events that you want to listen to, you can add via the connection.addEventCodeListener() method.

Bots LongPoll API #backtotop

Recently, VKontakte has added support for LongPoll for groups. That is now all the events that are for the Callback API will work for LongPoll, but for Bots LongPoll. This greatly simplifies the creation of bots. After all, LongPoll works faster than the Callback API, which makes it now the leader in the VKontakte platforms for bots.

EasyVK supports Bots LongPoll API from version 0.4.
All methods, except the connection.addEventCodeListener() method, copied from User LongPoll API. Also, the Bots LongPoll API differs from the User LongPoll API in the event structure, so the events that are in the usual LongPoll API will not work for this, the names will differ + answer format.

Usage example


vk.bots.longpoll.connect({
    forGetLongPollServer: {
        group_id: "1"
    }
}).then(connection => {
    connection.debug(vkr => {
        console.log(vkr);
    }).on("message_new", (msg) => {
        console.log(msg);
    });
}).catch(errHandler);

Events

The structure of events in the Bots LongPoll API differs from the usual ones, you can read about them here. But handling events is no different.

Usage example


vk.bots.longpoll.connect({

Callback API #backtotop

The Callback API is supported in EasyVK with a release of 0.3. Since then, the Callback API has been refactored 2 times and now works stably. Tests were conducted on the Heroku hosting server.

What is it?

The Callback API first allows you to listen to the events of your group: comments, posts, joins the group, various other interactions, also messages, etc. All the events you can see on the management page of your community. And more about how you can configure the Callback API read on this page.

I'll just tell you how to use it on an already configured server.

That's all. Configure the server, install node.js, npm, easyvk, and then follow the documentation below.

callbackAPI.listen(...)

Callback API works by the principle of "listen". You install the server, write the software and then the server accepts POST requests from th VK server. VK usually sends requests of a different nature: events, confirmations and others. Your server listens to all this and performs actions based on events. What events you want to listen to - it's your thing, it's configured in the API settings (group management). Callback API uses a express module to create a server and standard http module.

Characteristic Value Description
@returns {Promise} Standard object Promise, in a resolve function sends a cb object, there are variables connection, app and server
@param cbParams {Object} Group settings, list of groups whose events you want to listen to. Well, the server port. Details are described below.

To listen to events, you must call this method with a single cbParams object, in the properties of which you must enter one of the following settings.

Parameter Description
groups An array of groups where each element is a Group object
groupId The ID of your group, this ID you take from the server confirmation instructions (manage group -> API -> Callback API)
confirmCode The confirmation code that the server must return during the confirmation request. It is also taken, like groupId from the server connection settings (group management)
secret Secret key for requests. If the query does not have a key, an error will occur. In fact, this is the password for accessing your Callback server, you set it in the group settings. Its presence is not necessary.
port HTTP port for the server, I recommend to install process.env.PORT || 3000

Usage example

Well and now for understanding - an example. 


vk.callbackAPI.listen({
    groupId: "111111",
    secret: "my_password_group",
    confirmCode: "confirm_code_from_setting_group",
    port: (process.env.PORT || 3000)
}).then((cb) => {
    console.log(cb.web.app) // express app
    console.log(cb.web.server) // http server
    console.log(cb.connection) // connection (as a matter of fact it is the listener of events)
}).catch(errHandler);

For several groups 


vk.callbackAPI.listen({
    groups: [{
        groupId: "111",
        confirmCode: "confirm_code_from_settings_group",
        secret: "my_password_group"
    }, {
        groupId: "1112",
        confirmCode: "confirm_code_from_settings_group",
        secret: "my_password_group"
    }],
    port: (process.env.PORT || 3000)
}).then((cb) => {}).catch(errHandler);

For security and for the durability of the data leakage of your users, I recommend setting a password for requests (preferably also md5), although by default this is not necessary.

For the sake of honesty, I will remind you that the Callback API does not require authorization, although you may have noticed that a vk variable for listening. In general, there is an even simpler method. 


easyVK.callbackAPI.listen(/*....*/).catch(errHandler);

Then the sense of using the object of anisation disappears. I recommend using this option, because there is no need for authorization, but since anything can change, I still left the Streaming API in the initialization object. However, the second method does not mean that the Streaming API is a static utility. No. She just has other purposes. That's all.

Events

As in LongPoll, in Callback API there are events, you can listen to them in the same way. Through a connection variable. List of events you can look at this page (official documentation).

Usage example


vk.callbackAPI.listen({
    groupId: "111111",
    secret: "my_password_group",
    confirmCode: "confirm_code_from_setting_group",
    port: (process.env.PORT || 3000)
}).then((cb) => {
    cb.connection.on("message_new", (event) => {
        console.log(event);
        /*
            To find out which group the event belongs to,
            use it
        */
        console.log(event.group_id);
    });
}).catch(errHandler);

Sync execution?

This question often arises, because at the time of writing the code there are many nestings and the programmer starts to get confused in the code, what follows, where the output, where the input, etc. Of course, not everything needs to be done synchronously, but the creation of asynchronous code introduces some understanding in the essence of the program. Che tut (rus). Just give an example.
You have a need to make a condition, something like: if a person is a friend of this person, or he is subscribed to it, then do it. This is how it looks in code with asynchronous program execution. 


let params = [1, 356607530];
vk.helpers.isFriend(...params).then(isFriend => {
    if (isFriend) return true;
    else {
        return vk.helpers.userFollowed(...params).then(userFollowed => {
            return userFollowed;
        });
    }
}).then(isAdded => {
    if (isAdded) {
        console.log("You have added it as a friend!");
    } else {
        console.log(`Points are not added, you did not add it to friends!
         Try again...`);
    }
}).catch(errHandler);

Gromozdo, is not it (I'm sure you can be offered an option even more, or on a ball, even less, more thoughtful .. But I'm not an olymp, sorry)? It can be simpler, but already make all requests synchronously. 


easyVK({/*...*/}).then(async (vk) => {
    let params = [1, 356607530];
    let isFriend = await (vk.helpers.isFriend(...params));
    let userFollowed = await (vk.helpers.userFollowed)(...params);
    if (isFriend || userFollowed) {
        console.log("You have added it as a friend!");
    } else {
        console.log(`Points are not added, you did not add it to friends!
         Try again...`);
    }
}).catch(errHandler);

Of course, this example is not the best, because the first code can be placed in general in 11 lines, if desired, and the implementation in the second code is done so that in any case 2 requests are executed, and in fact there is no meaning, there will be more load.
